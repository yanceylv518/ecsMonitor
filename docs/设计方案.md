# 阿里云 ECS 监控桌面程序 — 设计方案

> 版本：v2（桌面程序版）
> 日期：2026-07-09
> 技术栈：Python + PySide6 + SQLite，目标平台 Windows

## 1. 概述

一个 Windows 桌面应用（PySide6/Qt），启动后在后台定时调用阿里云云监控 API 拉取 ECS 实例监控指标，写入本地 SQLite，界面上实时展示各实例的指标曲线和状态总览。支持最小化到系统托盘常驻运行。

## 2. 数据源与接口选型

| 用途 | 接口 | 说明 |
|------|------|------|
| 指标采集（核心） | 云监控 CMS `DescribeMetricList`，命名空间 `acs_ecs_dashboard` | CPU、网络、磁盘 IO 免装 agent 即有；实例装云监控插件后还有内存、磁盘使用率等 OS 级指标 |
| 实例发现 | ECS `DescribeInstances` | 自动发现地域下实例列表 + 名称/状态等元信息，带分页 |

不选 ECS 原生 `DescribeInstanceMonitorData` 的原因：指标少（无内存/磁盘使用率），且需逐实例调用。

**默认采集指标**（设置界面可勾选增删）：

| 指标名 | 含义 | 是否需要云监控插件 |
|--------|------|:--:|
| `CPUUtilization` | CPU 使用率 | 否 |
| `memory_usedutilization` | 内存使用率 | 是 |
| `diskusage_utilization` | 磁盘使用率 | 是 |
| `InternetInRate` / `InternetOutRate` | 公网入/出带宽 | 否 |
| `IntranetInRate` / `IntranetOutRate` | 内网入/出带宽 | 否 |
| `DiskReadBPS` / `DiskWriteBPS` | 磁盘读写吞吐 | 否 |

## 3. 整体架构

```
┌────────────────────────  GUI 主线程 (Qt)  ───────────────────────┐
│  主窗口                                                          │
│  ├─ 总览页（实例卡片：名称/状态/CPU/内存 最新值）                     │
│  ├─ 详情页（选实例 → pyqtgraph 多指标历史曲线）                      │
│  ├─ 设置页（AccessKey/地域/实例/指标/间隔/保留期）                   │
│  └─ 系统托盘（最小化常驻、气泡通知、退出）                            │
└──────────────────────────────┬──────────────────────────────────┘
                       Qt 信号/槽（跨线程通信）
┌──────────────────────────────┴──────────────────────────────────┐
│  采集线程 (QThread)                                               │
│  Scheduler → Discovery(DescribeInstances)                        │
│            → Collector(DescribeMetricList) → Storage(SQLite)     │
│            → 发射 data_updated 信号通知界面刷新                     │
└──────────────────────────────────────────────────────────────────┘
```

### 线程模型要点

- API 调用和入库全部放在 `QThread` 采集线程里，绝不阻塞 UI 线程
- 采集线程完成一轮后发射信号（`data_updated`、`collect_error`），UI 通过槽函数刷新图表和状态栏
- SQLite 连接不跨线程共享：采集线程持有写连接，UI 线程按需开只读连接查询历史数据
- 修改设置后通过信号通知采集线程热更新（无需重启程序）

## 4. 界面设计

主窗口采用侧边导航 + 三个页面：

1. **总览页** — 实例卡片网格。每张卡片显示：实例名/ID、运行状态、CPU%、内存%、公网出带宽的最新值；异常值（如 CPU > 90%）标红。双击卡片跳转详情页。
2. **详情页** — 顶部选实例 + 时间范围（1h / 6h / 24h / 7d），下方 pyqtgraph 绘制分组子图（CPU/内存一组、网络一组、磁盘一组），鼠标悬浮显示数值。
3. **设置页** — 分组表单：
   - 凭证：AccessKey ID / Secret（Secret 经 `keyring` 存入 Windows 凭据管理器，不落明文）
   - 采集：地域、实例（自动发现 or 手动勾选）、指标勾选、采集间隔、数据保留天数
   - 「测试连接」按钮：验证凭证有效性

**系统托盘**：关闭窗口 = 最小化到托盘，采集继续；托盘右键菜单：打开主界面 / 暂停采集 / 退出；可选指标超阈值气泡通知。

## 5. 数据与配置存储

### 5.1 存储位置（Windows 用户目录）

```
%APPDATA%\EcsMonitor\
├── config.json        # 界面设置（不含 AccessKey Secret）
├── ecs_metrics.db     # 指标数据（SQLite）
└── logs\              # 运行日志（按天滚动）
```

AccessKey Secret 存 Windows 凭据管理器（`keyring` 库），配置文件里只留 Key ID。

### 5.2 数据库表结构

```sql
-- 指标数据表
CREATE TABLE metrics (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    instance_id TEXT NOT NULL,          -- ECS 实例 ID
    metric_name TEXT NOT NULL,          -- 指标名，如 CPUUtilization
    timestamp   INTEGER NOT NULL,       -- 数据点时间（毫秒时间戳，API 原样）
    average     REAL,                   -- 周期内平均值
    maximum     REAL,
    minimum     REAL,
    created_at  TEXT DEFAULT (datetime('now'))
);
-- 去重：同一实例同一指标同一时间点只存一条
CREATE UNIQUE INDEX ux_metrics ON metrics(instance_id, metric_name, timestamp);
-- 查询加速：按实例+指标+时间范围查
CREATE INDEX ix_metrics_query ON metrics(instance_id, metric_name, timestamp DESC);

-- 实例元信息表
CREATE TABLE instances (
    instance_id   TEXT PRIMARY KEY,
    instance_name TEXT,
    region_id     TEXT,
    status        TEXT,
    updated_at    TEXT
);
```

数据保留：每天清理一次超过保留期（默认 30 天）的数据，防止数据库无限膨胀。

### 5.3 配置项（config.json）

```jsonc
{
  "region_id": "cn-hangzhou",
  "access_key_id": "LTAI...",        // Secret 不在此文件
  "instance_ids": [],                 // 空 = 自动发现
  "interval_seconds": 60,             // 采集间隔
  "period": 60,                       // 指标聚合粒度（秒）
  "lookback_seconds": 300,            // 回看窗口，与上轮重叠防漏点
  "retention_days": 30,
  "metrics": ["CPUUtilization", "memory_usedutilization", "..."],
  "cpu_alert_threshold": 90,          // 卡片标红阈值
  "start_minimized": false,
  "auto_start": false                 // 开机自启（HKCU\...\Run）
}
```

## 6. 核心流程（每个采集周期）

1. 按需刷新实例列表（每 10 个周期一次）→ 更新 `instances` 表
2. 按「批实例 × 指标」调用 `DescribeMetricList`（Dimensions 一次传多实例，每批 ≤ 50，调用量从「实例数 × 指标数」降为「批数 × 指标数」），时间窗口 `[now - lookback, now]`
3. 解析 Datapoints（JSON 字符串），处理 NextToken 分页 → 批量 `INSERT OR IGNORE` 入库
4. 发射 `data_updated` 信号 → UI 刷新总览卡片和当前打开的曲线图
5. 失败项指数退避重试 2~3 次，仍失败记日志并在状态栏提示，不中断整轮采集

## 9. 打包分发（Windows）

- **PyInstaller** `--onedir` 模式（比 `--onefile` 启动快，PySide6 体积大更适合），产出 `EcsMonitor/` 目录 + `EcsMonitor.exe`，预计 80~150MB
- 图标、版本信息写入 spec 文件；可选用 **Inno Setup** 做安装向导
- PyInstaller 不支持交叉编译：Linux 上开发调试，出 Windows 包需在 Windows 机器或 CI（GitHub Actions windows runner）上执行

## 10. 可靠性与安全

- RAM 子账号只读授权：`AliyunCloudMonitorReadOnlyAccess` + `AliyunECSReadOnlyAccess`
- Secret 不落明文文件，走系统凭据管理器
- API 限流退避重试；单点失败不影响整轮采集
- UTC 毫秒时间戳存储，界面按本地时区展示
- 开机自启（可选，写注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`）
